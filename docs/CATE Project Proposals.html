Description:
Joint supervision with Abdul Alim

As middleboxes are widely widely used in the Internet, a number of programmable
middlebox platforms have been developed over the last few years. Since middleboxes
mainly process network packets, programmable middleboxes requires high speed packet
I/O to process packets at line rate. Several high speed packet I/O systems (e.g.,
netmap, DPDK, Packet Shader) have been emerged that can receive/transmit network
packets directly from/to the user-space bypassing the OS kernel. These systems
provides C APIs for writing middlebox applications and are not usable from Java
applications.

However, many high performance computing projects have been using Java and achieved
comparable performance to C/C++ based systems. So it is also possible to write
middlebox applications in Java that could run at line rate. The main obstacle
to high speed packet processing in Java is that there is no packet I/O system
that can send/receive network packets directly from/to JVM memory. Once we have
such a packet I/O system, middlebox applications can be written in Java that
will run at line rate.

Project Goals

The main objective of this project is to implement a high speed packet I/O system
in Java on top of netmap/DPDK.

Roadmap

In order to implement such a system, we have to exploit two features of Java:
off heap Java memory using Unsafe class or ByteBuffer's allocate direct and
Java Native Interface (JNI) to interact with netmap/DPDK system. Implement
a few middlebox applications such as a firewall and a NAT in Java and compare
their performance with their C/C++ counter part.

Skills

1. Strong Java and C programming experience including network programming.
2. Good understanding of networking protocols.
